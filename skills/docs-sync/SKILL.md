# 문서 동기화 모드

> AI가 외부 기억을 활용하게 만드는 장치.

## 설계 철학

- AI는 기억을 못한다 → **외부 문서가 기억을 대신한다**
- 계획은 문서로 저장하고 새 세션에서 읽는다
- "작업 시작"과 "작업 마무리"에만 사용하는 의식(ritual)

---

## 사용법

| 호출 | 동작 |
|------|------|
| `/문서` | before + after 순차 실행 |
| `/문서 before` | 작업 전 문서 확인 (맥락 복원) |
| `/문서 after` | 작업 후 문서 갱신 (기억 저장) |

---

## Phase 1: Before — 맥락 복원

작업 시작 전 아래 문서를 **Read**하여 현재 상태를 복원한다.

### 필수 확인 문서

| 우선순위 | 문서 | 복원 내용 |
|---------|------|----------|
| 1 | `docs/AI_HANDOFF.md` | 이전 세션의 작업 상태, 미완료 작업, 주의사항 |
| 2 | `docs/AI_TASKS.md` | 전체 태스크 목록, 현재 Phase, 미완료 항목 |
| 3 | `docs/AI_CONTEXT.md` | 아키텍처, 임계값 레지스트리, Golden Flows |

> 위 파일이 없는 프로젝트는 `docs/` 내 존재하는 문서를 탐색하여 컨텍스트를 파악한다.

### 상황별 추가 확인

| 작업 유형 | 추가 확인 문서 |
|----------|--------------|
| UI/Compose 작업 | `docs/COMPOSABLE_MAP.md` |
| 네비게이션/라우팅 | `docs/APP_MAP.md` |
| 패키지 구조 변경 | `docs/ARCHITECTURE.md` |

### Before 완료 시 출력

```
📋 맥락 복원 완료
- 현재 상태: {AI_HANDOFF 요약}
- 진행 중 태스크: {AI_TASKS에서 미완료 항목}
- 주의사항: {있으면 표시}
- 이어서 할 작업: {추천}
```

## Phase 2: After — 기억 저장

작업 완료 후 결정 사항과 변경 내역을 문서에 기록한다.

### 갱신 판단

```bash
git diff --stat   # 변경된 파일 목록 확인
git log --oneline -5   # 최근 커밋 확인
```

### 문서별 갱신 규칙

| 문서 | 갱신 조건 | 기록 내용 |
|------|----------|----------|
| **AI_HANDOFF.md** | **항상** | 현재 브랜치, 마지막 커밋, 진행 중 작업, 주의사항 |
| **DEVELOPMENT_LOG.md** | **항상** | 오늘 날짜 섹션에 작업 내역 추가 |
| **AI_TASKS.md** | 태스크 완료/추가 시 | 체크박스 업데이트, 완료 날짜, 새 태스크 |
| **CHANGELOG.md** | 기능 추가/버그 수정 시 | Unreleased 섹션에 항목 추가 |
| **AI_CONTEXT.md** | 아키텍처/임계값 변경 시 | 변경된 구조, 새 패턴, 수치 갱신 |
| **ARCHITECTURE.md** | 패키지/파일 구조 변경 시 | 파일 트리, 모듈 설명 갱신 |
| **COMPOSABLE_MAP.md** | Composable 추가/삭제/변경 시 | 함수 트리 + 테이블 갱신 |

> 프로젝트에 없는 문서는 무시한다. 변경 없는 문서는 건드리지 않는다.

### 갱신 원칙

1. **사실만 기록한다** — 추측이나 계획이 아닌 실제 변경 사항
2. **기존 형식을 유지한다** — 각 문서의 기존 헤더/구조/스타일
3. **결정 사항을 명시한다** — 왜 이 방식을 선택했는지 (미래의 AI를 위해)

### After 완료 시 출력

```
📝 기억 저장 완료
- 갱신: {갱신된 문서 목록}
- 스킵: {변경 없어서 스킵한 문서}
- 다음 세션 참고: {중요 결정 사항 1~2개}
```

> 갱신된 문서 파일은 자동 커밋하지 않는다. 필요하면 `/커밋`을 별도 호출.

---

## 핵심: 미래의 AI를 위한 기록

After에서 가장 중요한 것은 **"왜"를 기록하는 것**:

- ❌ "DateUtils에 함수 추가" (what)
- ✅ "달력 월과 커스텀 월의 차이로 인한 버그. getEffectiveCurrentMonth()로 실효 월 개념 도입" (why)

이 기록이 다음 세션에서 `/문서 before` 시 맥락을 복원하는 핵심이 된다.
